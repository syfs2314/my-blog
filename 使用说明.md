# 博客修改指南

这份文档说明如何修改博客的各项内容。每次修改后，把改动 push 到 GitHub，Pages 会自动更新。

---

## 改博客名称、作者、简介

编辑 `_config.yml`：

```yaml
title: 我的个人博客        # 博客名称
author: Your Name         # 作者名（显示在页面底部）
description: 记录技术、学习和生活  # 简介（显示在首页）
```

---

## 改"关于我"页面

编辑 `about.md`，用 Markdown 格式直接写内容即可。

---

## 写新文章

在 `_posts` 目录下新建一个文件，文件名格式：`年-月-日-标题.md`

文件内容顶部必须包含以下 front matter：

```markdown
---
layout: default
title: 文章标题
date: 2026-05-10
---

正文从这里开始，使用 Markdown 格式编写。
```

示例 — 新建 `_posts/2026-05-10-学习笔记.md`：

```markdown
---
layout: default
title: 学习笔记
date: 2026-05-10
---

## 今天学了什么

- HTML 标签
- CSS 布局
- Jekyll 模板语法
```

首页会自动显示最近 5 篇文章，无需手动更新。

---

## 改样式

编辑 `assets/style.css`。常用调整：

| 想改的地方 | CSS 选择器 |
|-----------|-----------|
| 页面宽度 | `body` → `max-width` |
| 正文字号 | `body` → `font-size` |
| 标题大小 | `h1`, `h2`, `h3` |
| 链接颜色 | `a` → `color` |
| 日期颜色 | `.date`, `.post-date` → `color` |

---

## 改导航和页面结构

编辑 `_layouts/default.html`。这是所有页面的公共框架，包含 `<head>`、页头导航、页脚。

---

## 仓库改名或迁移

如果你的仓库名发生变化，编辑 `_config.yml` 中的 `baseurl`：

```yaml
# 仓库名是 用户名.github.io → 留空
baseurl: ""

# 仓库名是其他名称（如 my-blog） → 填 /仓库名
baseurl: /my-blog
```

---

## 文件对应关系总结

| 要改什么 | 改哪个文件 |
|---------|-----------|
| 博客名字、作者、简介 | `_config.yml` |
| 首页文章列表 | 自动生成（只管写新文章即可） |
| 文章内容 | `_posts/` 目录下的 Markdown 文件 |
| 关于我页面 | `about.md` |
| 样式 | `assets/style.css` |
| 页面框架（导航、页脚） | `_layouts/default.html` |
